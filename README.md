# Lyric Storage

MP3 플레이어. 플레이리스트 관리와, 가사 텍스트에 타임스탬프를 직접 입력해 동기화하는 가사 편집 기능이 특징입니다. 저장 포맷은 표준 `.lrc`라 다른 플레이어와도 호환됩니다. 아이콘은 전부 [Feather Icons](https://feathericons.com)를 사용합니다. Flask로 만든 웹 앱입니다.

## 실행

```bash
uv run web.py
```

기본적으로 http://127.0.0.1:5000 에서 열립니다. 단일 사용자 로컬/자체 호스팅용으로, 인증 없이 개발 서버로 동작합니다.

## 기능

- **재생**: play/pause, 이전/다음, seek, 볼륨, 셔플, 반복(전체/한 곡)
- **지원 포맷**: mp3, wav, m4a
- **플레이리스트**: "기본 플레이리스트"는 곡의 글로벌 라이브러리 역할을 합니다. 파일/폴더 추가(태그는 mutagen으로 자동 인식)는 기본 플레이리스트에서만 하고, 그 외 플레이리스트는 기본 플레이리스트에 등록된 곡 중에서 골라 추가합니다. 추가되는 음원은 내용 해시를 이름으로 `./data/songs/`에 복사되어 보관됩니다(원본 경로가 바뀌거나 삭제돼도 라이브러리에는 영향 없음, 동일 파일 중복 추가 시 재사용). 드래그로 순서 변경, 여러 개의 플레이리스트 생성/전환, 마지막 세션 자동 복원도 지원합니다.
- **가사 편집**: "가사 편집" 탭에서 각 줄마다 가사 텍스트와 타임스탬프(분:초:밀리초, 예 `1:12:500`)를 직접 입력/수정하면 자동으로 저장됩니다. 마크다운 서식(`**굵게**`, `*기울임*`)과, 번역 등 보조 줄(`> `로 시작)을 지원합니다. 가사는 음원 파일 옆에 `.lrc`로 저장되고, "가사 보기" 탭에서 재생 시 현재 줄이 자동으로 하이라이트됩니다.

## 데이터 저장 위치

- 플레이리스트/설정: 프로젝트 폴더의 `./data/`
- 음원 파일: `./data/songs/<내용 해시>.<확장자>` (라이브러리에 추가 시 원본을 복사)
- 가사: 음원 파일과 같은 폴더(`./data/songs/`)에 `.lrc` (쓰기 불가 시 `./data/lyrics_cache`로 폴백)
